Gestão de agentes de IA para Telegram, focado em simplicidade, facilidade de uso e segurança.

Rode múltiplos agentes independentes numa única VPS, cada um com personalidade própria, memória persistente, ferramentas modulares e subagentes especializados.

Setup Rápido
git clone https://github.com/alozs/smb-claw
cd smb-claw && bash setup.sh

Por que o SMB Claw existe?

Depois de semanas passando raiva com o OpenClaw — que às vezes funciona, às vezes não, você pede uma coisa e ele faz outra — decidi criar minha própria alternativa com uma proposta diferente: extremamente simples, eficiente e funcionando como um verdadeiro assistente de IA confiável.

Sem precisar lidar com infraestrutura complexa como Kubernetes. Basicamente um script e você já tem um agente rodando — do jeito que deveria ser desde o começo. Funciona tanto em VPS com systemd quanto em containers Docker.

Diferença SMB Claw Outros (OpenClaw)
RAM por Agente ~75 MB 8–16 GB (Cluster)
Setup Inicial ~2 minutos 15–60 minutos
Complexidade ~11k linhas ~350k linhas
Infraestrutura VPS $5/mês VPS $20+/mês

01. Terminal

Use apenas para o setup inicial. Detecta Claude CLI, Codex, OpenRouter e configura tudo automaticamente.

02. Telegram

Crie, configure e apague agentes via chat com o comando /criar_agente. Gestão total pelo celular.

03. Painel Web

Dashboard para logs em tempo real e ajustes finos. Acesso seguro via token temporário (30 min) gerado no bot.

Acesso Temporário

Painel protegido por tokens in-memory. Sem senhas estáticas. Expiração automática após 30 minutos de inatividade.

Sub-agentes

Delegue tarefas para agentes especializados criados dinamicamente via chat.

Memória em Camadas

Gestão eficiente de contexto: Soul (regras imutáveis), USER.md (preferências) e MEMORY.md (longo prazo). Persistência técnica via SQLite que sobrevive a reinicializações sem perda de foco.

Bug Fixer Autônomo

Agente que monitora logs, detecta erros e sugere correções via Telegram de forma proativa.
